## Project Overview

This project delivers two connected marketing assets for a Saudi Arabian e-commerce store (NahamTech) selling ZKTeco biometric and POS products:

1. **A Google Search Campaign** — structured ad groups with exact-match keywords and ad copy for ZKTeco product lines.
2. **A Google Merchant Center (GMC) Product Feed** — a clean, GMC-compliant product feed built by merging two disorganized source files.

Both outputs are designed for the Saudi market with prices in SAR and Arabic + English keyword coverage.

## Phase 1 — Google Search Campaign

### Strategy

The campaign targets high-intent buyers in Saudi Arabia searching for specific ZKTeco products. The structure follows Google Ads best practices:

- **Single Keyword Ad Groups (SKAGs) hybrid** — each ad group is tightly themed around one product or category
- **Exact match only** — eliminates wasted spend and ensures high relevance scores
- **Bilingual keywords** — Arabic and English variants to capture both search behaviors in KSA

### Why these ad groups?

- **Model-specific ad groups** (MB2000, UFace800, SpeedFace V5L, F18, F22, MB10) — these are the highest-value products with strong search intent. Each model gets its own ad group so the ad copy can include exact pricing and features.
- **Category ad groups** (General attendance, Access Control, USB Readers, Biosecurity) — capture broader funnel traffic that hasn't decided on a specific model yet.
- **BioTime Software** — frequently searched by existing ZKTeco hardware owners looking for software installation/support, creating upsell and service opportunities.

## Phase 2 — Google Merchant Center Feed

### Feed Specification

The feed strictly follows [Google's Product Data Specification](https://support.google.com/merchants/answer/7052112) with these 14 required/recommended columns in exact order:

```
id, title, description, link, image_link, availability, price,
brand, condition, gtin, mpn, identifier_exists,
google_product_category, product_type
```

### How Products Were Processed

1. **Extracted** 147 rows from the XLSX source file
2. **Filtered out** non-shoppable rows:
   - Service request forms ("طلب عرض سعر" / quote requests)
   - Consultation requests ("طلب استشارة")
   - Software licenses without a physical product
3. **Deduplicated** where the same physical product appeared at multiple price tiers — kept the representative listing with the most common availability
4. **Translated** Arabic availability values:
   - `متاح` → `in_stock`
   - `غير متاح` → `out_of_stock`
5. **Cleaned** HTML from descriptions — stripped tags and wrote concise English descriptions for GMC compliance
6. **Formatted prices** as `XXX.XX SAR` (all prices from the source are VAT-inclusive per Saudi regulations; 15% VAT is baked in)

### Image Matching Logic

Since there was no explicit mapping between products and image URLs, the matching was done by:

1. **Extracting model numbers/MPNs** from the product data (e.g. `MB2000`, `UFACE800`, `F18`, `ZD22042-T0EG00EZ`)
2. **Scanning image URLs** for matching slugs (e.g. `MB2000_1.png`, `uFace-800-1.jpg`, `F18_1.jpg`)
3. **Case-insensitive substring matching** with normalization (hyphens, underscores, spaces treated equally)
4. **First-match rule** — when a product matched multiple images, only the first (best) match was kept per the IMAGE RULE constraint

Products with zero matching images received an empty `image_link` field (1 product: Zebra USB cable CBA-U01-S07ZAR).

## Data Decisions & Rules Applied

| Rule                          | Implementation                                                                                                      |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Column order is strict**    | All output files follow the exact 14-column order specified                                                         |
| **One image per product**     | Only the single best-matching URL is in `image_link`; extras discarded                                              |
| **VAT-inclusive SAR pricing** | All prices formatted as `XXXX.XX SAR`. Saudi 15% VAT is already included in source prices                           |
| **Availability mapping**      | Arabic `متاح`/`غير متاح` converted to GMC-standard `in_stock`/`out_of_stock`                                        |
| **No GTIN**                   | Products lack universal barcodes, so `gtin` is empty and `identifier_exists` is set to `yes` when an MPN is present |
| **Brand normalization**       | Arabic+English brand names (e.g. "ZKTeco - زد كي تيكو") simplified to English-only for GMC compliance               |
| **Services excluded**         | 8 service/consultation rows removed — GMC requires physical or digital goods, not service inquiries                 |

## How to Use Each File

### Google Merchant Center Upload

1. Open `gmc_product_feed.csv` or `.xlsx`
2. Verify image URLs are accessible (they should resolve to `img.nahamtech.org`)
3. Upload via **Merchant Center > Products > Feeds > Add feed** (choose "Scheduled fetch" or "Upload")
4. Select `Saudi Arabia` as target country and `SAR` as currency

### Google Ads Editor Import

1. Open Google Ads Editor
2. Go to **Account > Import > From CSV**
3. Select `google_ads_campaign.csv`
4. Map columns: Ad Group, Keyword, Match Type, Headlines, Descriptions
5. Review and post changes

### Programmatic / API Use

- Use the `.json` files for integration with custom dashboards, automation scripts, or feed management platforms
- JSON structure is a flat array of objects keyed by column name
